🚨 TROUBLESHOOTING

Android VAPID Key
파싱 오류 해결

Web Push 구독 실패 — Android Chromium 크로스브라우징 이슈

⚠ 원인 (CAUSE)
Android 환경에서 VAPID Public Key를 Base64 문자열 그대로 전달 시 브라우저 내부 파싱 오류로 Web Push 구독 실패.
iOS와 달리 Android Chromium이 Uint8Array 바이너리 배열을 엄격하게 요구.
✓ 해결 (FIX)
Base64 URL-safe 문자열 → Uint8Array 변환 유틸 함수를 직접 구현하여 applicationServerKey 인자로 주입.
urlBase64ToUint8Array(publicKey) 함수를 packages/shared에 등록하여 재사용 구조 확보.
◎ 결과 (RESULT)
Android / iOS / Desktop 크로스 브라우징 Web Push 구독 정상 동작 확인.
shared 유틸로 추출하여 이후 다른 플랫폼 확장 시 코드 중복 없이 재사용 가능한 구조 확보.

Cloudflare Tunnel
SSL/TLS 프로토콜 누락

API 통신 전체가 침묵하며 실패 — 환경변수 URL 파싱 문제

⚠ 원인 (CAUSE)
Cloudflare Tunnel 배포 환경에서 환경변수에 저장된 API 엔드포인트 주소에 https:// 프로토콜이 누락됨.
TLS 인증 메커니즘이 강제 적용된 CF Tunnel의 특성상 일반 HTTP 요청을 거부하여 API 통신 전체가 침묵하며 실패.
✓ 해결 (FIX)
Network 탭과 서버 로그를 교차 분석하여 요청 자체가 브라우저 단에서 차단됨을 확인.
환경변수 URL 앞에 https:// 명시 후, Tunnel 재시작 시마다 주소 갱신이 필요한 구조적 문제를 팀 공유 문서로 기록하여 온보딩 비용 제거.
◎ 결과 (RESULT)
API 통신 정상화. 이후 팀 전원이 같은 실수를 반복하지 않도록 체크리스트 문서화 — 디버깅 경험을 팀 자산으로 전환.

HTML5 CSS3 JavaScript PokeAPI CF Pages
🛠 Key Engineering Points
  • 개발 착수 전 상세 워크플로우(흐름도) 선제 설계 — 예외 처리 분기 미리 확보
  • 이중 속성 조합 상성 배수 연산 알고리즘 직접 구현
  • 외부 API 미제공 데이터를 위한 정형화된 Mock API JSON 스키마 설계 및 바인딩
  • Cloudflare Pages 정적 배포로 무비용 CDN 서빙
💡 ARCHITECTURE

Mock API &
알고리즘 설계

외부 API 의존성을 줄인 독립적 데이터 레이어 구축

📐 설계 (DESIGN)
PokeAPI가 제공하지 않는 이중 속성 상성 테이블을 직접 정형화된 JSON 스키마로 설계. 외부 API 변경에 무관하게 독립적으로 동작하는 Mock API 레이어 구축.
⚙ 알고리즘 (ALGORITHM)
포켓몬의 두 타입 조합 시 각 방어 타입 배수를 곱산하는 연산 로직 직접 구현.
예: 풀+독 타입 vs 불 공격 = 2× (약점) × 1× = 최종 2배 계산.
◎ 성과 (RESULT)
외부 API 없이 오프라인에서도 완전 동작하는 독립 데이터 레이어 확보. Cloudflare Pages 정적 배포로 무비용 CDN 서빙 중.
